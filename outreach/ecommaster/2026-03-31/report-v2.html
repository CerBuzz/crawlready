07 — Recomendaciones

Qué arreglar, por impacto

1
Server-Side Rendering (SSR) o pre-rendering del contenido
Este es el bloqueo fundamental. Ecommaster usa WordPress con Elementor, que renderiza todo el contenido vía JavaScript. Activar SSR, pre-rendering estático o un plugin de caché que sirva HTML completo haría visible todo el contenido instantáneamente para agentes IA.
Crítico
2
Añadir datos estructurados (Organization, Course, Product)
Incluso con SSR, los datos estructurados JSON-LD son el canal más directo para que los agentes IA comprendan el negocio. Schema Organization con identidad, Course para cada máster, y Product con precios (€3.500–3.900).
Impacto alto
3
Meta tags y Open Graph en todas las páginas
Título, descripción y OG tags deben estar presentes en el HTML inicial (antes de JavaScript). Esto permite que los agentes IA entiendan cada página desde los resultados de búsqueda.
Impacto alto
4
Asegurar contenido HTML legible sin JavaScript
Los textos clave — nombres de másteres, precios, números de contacto, descripción de la escuela — deben estar en el HTML que el servidor entrega. No depender exclusivamente de que el navegador ejecute JS para mostrarlos.
Crítico
5
Añadir archivo llms.txt
Un archivo llms.txt en la raíz del dominio proporciona un resumen estructurado del negocio específicamente diseñado para modelos de lenguaje. Es rápido de implementar y ofrece un canal directo con agentes IA.
Impacto medio
