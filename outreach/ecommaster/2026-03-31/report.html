07 Recomendaciones
1. Resolver el renderizado JavaScript (causa raíz) Impacto crítico
Este es el problema #1 y resuelve el 80% del informe. Elementor renderiza todo el contenido vía JS, lo que significa que un agente IA recibe HTML vacío. Solución: habilitar Server-Side Rendering (SSR) o pre-renderizado estático del contenido. Alternativas: (a) usar un plugin de caché que genere HTML estático (WP Super Cache, W3 Total Cache con page cache), (b) configurar Elementor para pre-renderizar contenido crítico, (c) implementar un servicio de pre-rendering como Prerender.io. Una vez resuelto, el contenido existente se vuelve visible automáticamente.
2. Añadir datos estructurados (Course, Organization) Impacto alto
Una vez el contenido sea legible, añadir schema JSON-LD: Organization (nombre, fundadores, ubicación), Course para cada máster (nombre, descripción, provider, offers con precio €3.500–€3.900, duration, format). Esto permite a los agentes IA listar y comparar los másteres de Ecommaster con la competencia.
3. Crear un archivo llms.txt Impacto alto
Mientras se resuelve el renderizado JS, un archivo /llms.txt sirve como canal directo de comunicación con LLMs. Contenido: quién es Ecommaster, qué másteres ofrece, precios, cómo matricularse, diferenciadores (15 años, 190+ profesores, 1.200+ alumnos). Implementación inmediata: 15 minutos. Efecto: un agente IA que acceda a /llms.txt obtendría toda la información que hoy es invisible.
4. Exponer información de contacto en HTML estático Impacto medio
Email, teléfono y enlace WhatsApp deben estar presentes en HTML puro (no inyectados por JS). Una etiqueta <a href="mailto:..."> y <a href="tel:..."> en el footer HTML garantiza que cualquier agente IA pueda ofrecer canales de contacto al usuario.
5. Asegurar que los formularios existan en HTML Impacto medio
Contact Form 7 genera formularios HTML por defecto, pero si se cargan dentro de widgets Elementor que dependen de JS, el formulario desaparece del HTML estático. Verificar que al menos el formulario de contacto principal se renderice en HTML server-side.
6. Publicar precios en HTML accesible Impacto medio
Los precios (Máster Online €3.500, Máster Marketplaces €3.900, financiación en 5 cuotas) deben ser legibles directamente en HTML. Un agente IA que compara opciones de formación en ecommerce descartará Ecommaster si no puede ver el precio, aunque sea competitivo.
El problema es grave, pero la solución es concreta
Ecommaster tiene 15 años de reputación, 190+ profesores y 1.200+ alumnos. Solo necesita que los agentes IA puedan verlo. Una causa raíz (renderizado JS), una solución (pre-rendering + datos estructurados).
